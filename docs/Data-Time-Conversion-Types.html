data TimeReader Source #

Determines how to read a time string.

Since: 0.1

Constructors

MkTimeReader 

Fields

  • format :: TimeFormat

    Format used when parsing the time string.

    Since: 0.1

  • srcTZ :: Maybe SrcTZ

    Timezone in which to read the string. Nothing corresponds to local timezone. If srcTZ is SrcTZDatabase then:

    • timeString should not contain a timezone like EST.
    • format should not mention the timezone option %Z.

    Since: 0.1

  • locale :: TimeLocale

    Locale to use when parsing.

    Since: 0.1

  • today :: Bool

    True if this is the current day. This is a convenient way to parse the string as if it includes the current day's date.

    Since: 0.1

  • timeString :: Text

    The time string to parse.

    Since: 0.1

defaultTimeReader :: Text -> TimeReader Source #

Given a time string, returns a default time reader.

  • format = "%H:%M": 24hr hours:minutes
  • srzTZ = Nothing: local
  • locale = timeLocaleAllZones: all locales
  • today = False: do not automatically add current day

Since: 0.1

data SrcTZ Source #

Determines what timezone we use when reading a time string.

Since: 0.1

Constructors

SrcTZDatabase TZDatabase

Use the TZDatabase to determine which timezone to use.

Since: 0.1

SrcTZLiteral

Timezones are not given any extra consideration. This is useful when either the string itself contains a timezone (e.g. "15:03 EST") or we want UTC.

data TZDatabase Source #

Options for interpreting a timezone.

Examples

Expand
>>> TZDatabaseLabel America__New_York
TZDatabaseLabel America__New_York

>>> TZDatabaseText "America/New_York"
TZDatabaseText "America/New_York"

Since: 0.1

Constructors

TZDatabaseLabel TZLabel

Uses a TZLabel. This option should be preferred as it enables total conversions.

Since: 0.1

TZDatabaseText Text

Interprets the Text as a tz database name e.g. America/New_York. Obviously such conversions cannot be total.

Since: 0.1
